# 代理工作流工作坊

欢迎来到 **代理工作流** 工作坊！在这个实践课程中，您将使用 GitHub Copilot 的代理工作流功能（`gh aw`）在您的 GitHub 仓库内直接自动化真实世界的任务——无需服务器，无需维护流水线。

## 工作坊概览

代理工作流允许您用自然语言描述 *您想做什么*。Copilot 会思考 *如何* 做到，运行必要的工具，并将结果发回 GitHub。在本工作坊中，您将引导工具、构建两个实用的工作流，并连接一个 ChatOps 斜杠命令——所有这些都从命令行完成。

### 学习目标

完成本工作坊后，您将能够：

1. **引导代理工作流** - 安装 `gh aw` 扩展并在仓库中初始化它，并为议题和拉取请求设置日常摘要。
2. **构建黑客新闻摘要** - 创建一个日常工作流，将相关的黑客新闻故事显示为 GitHub 议题。
3. **使用 ChatOps 模式** - 实现一个斜杠命令，在黑客新闻故事评论上运行情感分析并内联回复。

## 先决条件

参加本工作坊之前，请确保您有：

- [ ] GitHub 账户，拥有有效的 **Copilot Pro、Pro+、Business 或 Enterprise** 订阅
- [ ] **Git** 已安装并配置
- [ ] **GitHub CLI (`gh`)** 已安装并通过身份验证（`gh auth login`）
- [ ] 对终端的基本熟悉度

> [!NOTE]
> 如果您使用 Copilot Business 或 Copilot Enterprise，请确保您的组织管理员已在策略设置中启用了 **Copilot 扩展** 和 **代理工作流**。

## 成功的提示

1. **在提示中保持具体** - 您提供的上下文越多，代理的表现就越好。
2. **阅读生成的工作流文件** - 理解代理写的东西有助于您调整它。
3. **迭代** - 如果第一个结果不完全符合您的需要，添加约束条件并重新运行。
4. **检查它创建的议题/PR** - 代理工作流将其输出发送到 GitHub，所以留意新议题。
